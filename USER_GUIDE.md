## 1. 产品简介

**AI 文章润色助手** 是一款基于大语言模型的智能写作辅助工具，覆盖从基础润色到高级文本处理的完整工作流。

### 1.1 你能用它做什么

| 场景 | 说明 |
|------|------|
| 📝 **论文润色** | 学术风格的语法修正、逻辑优化、格式规范 |
| 💼 **商务文案** | 邮件、方案、汇报材料的话术优化与正式化 |
| 📱 **自媒体内容** | 公众号文章降重改写、去 AI 痕迹、通俗化 |
| ✍️ **日常写作** | 各类文章的风格转换、AI 续写、格式标准化 |

### 1.2 核心功能一览

- **6 大润色工具**：智能润色、降重改写、去AI痕迹、通俗化、AI续写、话术转换
- **5 种润色风格**：学术严谨、商务正式、自媒体活泼、简洁凝练、自定义
- **4 种角色人格**：资深编辑、语文老师、商务顾问、学术导师
- **3 级润色强度**：轻度（仅修错）、中度（优化语句）、深度（重构全文）
- **知识库增强 (RAG)**：基于 5 份 + 自定义写作规范资料库的专业建议
- **智能文本分析**：可读性评分、语病检测、重复度检测、敏感词筛查、关键词提取、摘要生成
- **4 种导出格式**：Markdown、纯文本、Word (.docx)、PDF
- **流式实时输出**：逐字生成，可随时中断
- **多轮对话**：上下文连贯的持续优化

## 2. 快速上手

### 2.1 环境要求

| 依赖 | 版本要求 |
|------|---------|
| Python | ≥ 3.10 |
| Node.js | ≥ 18.x（仅开发/构建时需要） |
| 操作系统 | Windows / macOS / Linux |

### 2.2 安装步骤

```bash
# 1. 克隆项目 / 解压项目到本地
cd polish-assistant

# 2. 创建 Python 虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
# source venv/bin/activate

# 4. 安装 Python 依赖
pip install -r requirements.txt

# 5. （可选）安装前端依赖并构建
cd frontend
npm install
npm run build
cd ..
```

### 2.3 配置 API Key

在项目根目录创建 `.env` 文件（可选，也可以通过前端界面配置）：

```env
# DeepSeek API（主模型，推荐）
DEEPSEEK_API_KEY=sk-your-deepseek-api-key
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

# Qwen API（备用）
QWEN_API_KEY=sk-your-qwen-api-key
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

> **安全提示**：`.env` 文件已在 `.gitignore` 中排除，不会被提交到 Git。你也可以不创建 `.env`，直接在应用内通过 API 配置弹窗输入 Key（仅保存在浏览器 sessionStorage 中，关闭浏览器即清除）。

### 2.4 启动服务

```bash
# 单服务模式（推荐）
python api_server.py
```

打开浏览器访问 **http://127.0.0.1:8000**

- 前端页面 → `http://127.0.0.1:8000`
- API 文档（Swagger）→ `http://127.0.0.1:8000/docs`

### 2.5 快速体验

1. 在输入框中粘贴一段需要润色的文字
2. 底部工具栏选择**润色工具**（如「智能润色」）和**风格**（如「学术严谨」）
3. 点击右侧**发送按钮**（或按 Enter 键）
4. 等待 AI 逐字生成润色结果，在聊天气泡中查看
5. 可继续发送指令优化（如「再简洁一些」「加入数据支撑」）

## 4. 六大润色工具详解

系统提供 6 种专用润色工具，通过输入栏左侧的 **工具选择器**（✨ 图标）切换：

### 4.1 智能润色 (Polish)

**基础润色工具**，支持三级强度控制。

| 强度 | 说明 | 适用场景 |
|:----:|------|----------|
| **轻度** | 仅修正语法错误、错别字、标点 | 已有较好基础，只需微调 |
| **中度** | 优化语句流畅度、调整措辞 | 需要一定加工润色 |
| **深度** | 重构段落结构、优化逻辑层次 | 大幅改写的深度优化 |

**参数**：强度 (light/medium/deep)、风格、角色、目标字数、模式（全文/逐段）

### 4.2 降重改写 (Paraphrase)

降低文本重复率，保持原意不变地改写。

**改写策略**：
- 同义词替换
- 主动 ↔ 被动语态转换
- 长句拆分 + 短句合并
- 语序调整

**核心原则**：**改形不改意** — 只改变表达方式，不改变核心观点。

**参数**：风格、角色、目标字数

### 4.3 去 AI 痕迹 (De-AI)

消除文本中的机器化特征，模拟真人手写风格。

**消除目标**：
1. 僵化句式（如「首先…其次…最后」的刻板结构）
2. 过度工整（每段长度完全一致）
3. 空洞修饰（「具有重要意义」「不可或缺」等套话）
4. 缺乏个性（语调单一、无情感变化）
5. 句式单一（连续句子结构相同）

**参数**：风格、角色

### 4.4 通俗化 (Simplify)

将专业、晦涩的内容转化为大众易懂的表达。

**简化策略**：
- 化繁为简（替换专业术语）
- 举例子（增加生活类比）
- 拆长句（每句不超过 25 字）
- 加解释（关键概念补充说明）
- 降密度（控制信息密度）

**参数**：风格、角色、目标字数

### 4.5 AI 续写 (Continue)

基于已有文本的风格和内容，自然续写后续段落。

**特点**：
- 保持原文风格一致
- 逻辑自然连贯
- 内容充实有据

**参数**：风格、角色、目标字数

### 4.6 话术转换 (Tone Shift)

改变文本的语气和表达风格，支持 5 种子类型：

| 子类型 | 说明 | 示例场景 |
|:------:|------|----------|
| **委婉表达** | 直接 → 委婉 | 拒绝邮件、负面反馈 |
| **感染力增强** | 平淡 → 有感染力 | 演讲稿、宣传文案 |
| **积极转化** | 消极 → 积极 | 问题表述转化为机会 |
| **正式化** | 口语 → 正式 | 报告、正式函件 |
| **口语化** | 正式 → 口语 | 社交媒体、聊天场景 |

## 6. 文本智能分析

通过顶部 Header 的 📊 **智能分析按钮**打开分析弹窗，从 6 个维度全面评估文本质量。

### 6.1 六维度分析

| 分析维度 | 说明 | 输出 |
|----------|------|------|
| **可读性评分** | 基于句长、罕见字比例、标点密度的中文可读性公式 | 0-100 分 + 难度等级 + 适合年级 |
| **语病检测** | 12 种常见中文语病正则模式匹配 | 错误描述 + 上下文 + 修改建议 |
| **重复度检测** | 4-gram 短语重复分析 | 重复短语 + 次数 + 位置 |
| **敏感词筛查** | 20+ 词库（广告违禁/不文明/绝对化用语） | 敏感词 + 风险提示 |
| **关键词提取** | TF-IDF 启发式 + 停用词过滤 | Top-K 关键词 + 词频 |
| **摘要生成** | 关键词匹配度排序 + 位置加权 | 自动摘要 |

### 6.2 可读性评分等级

| 分数 | 难度 | 适合年级 | 典型文本 |
|:----:|------|----------|----------|
| 90-100 | 非常容易 | 小学 | 儿童读物 |
| 70-89 | 容易 | 初中 | 通俗文章 |
| 50-69 | 中等 | 高中 | 一般文章 |
| 30-49 | 较难 | 大学 | 专业文章 |
| 0-29 | 困难 | 研究生+ | 学术论文 |

### 6.3 语病检测覆盖

系统可检测 12 种典型中文语病，包括：
- 「通过…使…」缺少主语
- 「之所以…的原因是因为…」句式冗余
- 「进行了…的…」动词弱化
- 「这是一个…的问题」句式冗余
- 语义重复（「凯旋归来」「免费赠送」）
- 搭配不当、成分残缺、句式杂糅等

### 6.4 使用方式

1. 在输入框输入或粘贴需要分析的文本
2. 点击 Header 中的 📊 图标
3. 弹窗自动分析并展示 6 个维度的结果
4. 点击各维度面板展开查看详情
5. 修改文本后点击「重新分析」更新结果

## 7. 知识库与语料库

### 7.1 知识库概述

系统内置 **5 份写作规范资料**（约 11,000 字），在润色时自动检索最相关的规范条文，增强润色质量和专业性。

| 资料名称 | 字数 | 内容范围 |
|----------|:----:|----------|
| 学术写作规范手册 | ~3,000 | 论文结构、引用规范、学术语言 |
| 商务邮件写作指南 | ~2,000 | 邮件格式、正式用语、沟通礼仪 |
| 新媒体写作技巧 | ~2,500 | 标题技巧、排版、传播策略 |
| 常见语法错误修正指南 | ~2,000 | 常见语病、错误用法纠正 |
| 中文标点符号使用规范 | ~1,500 | 标点标准用法 |

### 7.2 双引擎检索策略

系统使用创新的**双引擎检索**确保 100% 可用：

```
用户查询
    ↓
主引擎：ChromaDB + HuggingFace Embedding → 可用 → 向量检索 (Top-K=3)
    ↓ 不可用（无网络）
回退引擎：TF-IDF + BM25 纯 Python 实现 → 关键词检索 (Top-K=3)
```

即使在完全断网的环境下，知识库也能正常工作。

### 7.3 引用来源查看

润色完成后，切换到「**引用来源**」Tab 查看：
- 本次润色引用了哪些写作规范
- 每份参考资料的**相关度评分**（0-100% 进度条）
- 点击展开具体的**规范原文片段**

也可点击 AI 消息底部的「参考了 N 份知识库资料」按钮快速跳转。

### 7.4 自定义语料库

通过设置弹窗（点击侧边栏底部齿轮图标 → 语料库标签）管理自定义知识：

- **查看列表**：所有语料的文件名、字符数、大小、修改时间
- **添加语料**：填写标题 + 内容，保存为 `.txt` 自动加入知识库
- **删除语料**：选中语料，确认后删除
- **重新加载**：增删语料后，点击「重新加载」重建索引使其生效

## 9. API 配置与模型切换

### 9.1 支持的模型

点击侧边栏底部的**连接状态指示器**（绿色圆点或红色断开图标）打开 API 配置弹窗：

| 模型 | 标识 | 特点 | 推荐场景 |
|------|:----:|------|----------|
| **DeepSeek-V3** | deepseek | 性价比高，中文能力强 | **主力推荐** |
| **Qwen-Max** | qwen | 阿里通义千问，中文顶级 | 复杂中文任务 |
| **Doubao** | doubao | 字节跳动豆包 | 备选方案 |
| **自定义** | custom | 任意 OpenAI 兼容接口 | 灵活接入其他模型 |

### 9.2 配置步骤

1. 选择模型类型
2. 输入对应的 **API Key**
3. （自定义模型时）输入 **Base URL**
4. 点击「**测试连接**」验证连通性
5. 验证成功后点击保存

### 9.3 API Key 安全

- **优先级**：前端界面输入（内存）> `.env` 文件 > 默认值
- **前端缓存**：`sessionStorage`（页面刷新保留，关闭浏览器即清）
- **绝不落盘**：API Key 不写入任何文件，确保安全

### 9.4 高级参数

在「更多设置」中可调整：

| 参数 | 默认值 | 说明 |
|------|:------:|------|
| **Temperature** | 0.7 | 创造性控制（0.1 严谨 ↔ 1.5 创造） |
| **Max Tokens** | 4096 | 最大输出长度 |
| **知识库增强 (RAG)** | 开启 | 是否启用写作规范检索增强 |
| **润色模式** | 全文 | 全文润色 / 逐段润色 |

## 10. 多对话管理

### 10.1 对话操作

通过左侧边栏管理多个对话：

| 操作 | 按钮/方式 | 说明 |
|------|-----------|------|
| **新建对话** | ➕ 按钮 | 创建空白对话，自动清空后端上下文 |
| **切换对话** | 点击对话项 | 加载该对话的全部历史消息 |
| **搜索对话** | 顶部搜索框 | 按关键字过滤对话列表 |
| **清空对话** | 对话项右侧菜单 → 清空 | 删除对话内消息，保留对话本身 |
| **删除对话** | 对话项右侧菜单 → 删除 | 完全删除对话及所有消息 |

### 10.2 数据持久化

- 对话数据自动保存在浏览器 `localStorage` 中
- 刷新页面或重启浏览器后**对话不丢失**
- 切换/新建对话时自动清空后端上下文，避免跨对话污染

## 11. 常见问题 FAQ

### Q1: 启动时报错 `ModuleNotFoundError`

**A**: 确保已安装所有依赖：

```bash
pip install -r requirements.txt
```

如果某些包（如 `chromadb`、`sentence-transformers`）安装失败，不影响基本使用——系统会自动降级为 TF-IDF 本地检索引擎。

### Q2: 知识库引用来源始终显示 0

**A**: 
- 确认「更多设置」中「知识库增强 (RAG)」开关已**开启**
- 首次启动时 `docs/` 目录下的资料会被自动加载索引
- 如果网络不可用，系统自动使用本地 TF-IDF 引擎，同样会产生引用来源

### Q3: 为什么润色结果与预期不符？

**A**: 尝试以下调整：
- 选择更适合的**润色风格**和**角色**
- 调整**润色强度**（轻度改动更保守，深度改动更彻底）
- 在「更多设置」中调整 **Temperature**（越低越严谨，越高越有创造性）
- 使用**多轮对话**继续优化结果

### Q4: 前端页面无法访问

**A**: 
- 确认后端服务已启动：`python api_server.py`
- 确认端口 8000 未被占用
- 如果前端显示空白，需要重新构建：`cd frontend && npm run build`

### Q5: 如何进行开发调试？

**A**:

```bash
# 终端 1 — 启动后端
python api_server.py

# 终端 2 — 启动前端开发服务器（支持热更新）
cd frontend
npm run dev             # 访问 http://localhost:5173
```

### Q6: 文件上传支持哪些格式？

**A**: 支持 `.txt` 和 `.md` 格式，无文件大小限制（长文自动处理）。

### Q7: 对话数据存储在哪里？

**A**: 浏览器 `localStorage`，key 为 `polish-assistant-conversations`。清除浏览器数据会导致对话丢失，建议及时导出重要结果。

### Q8: 如何实现完全离线使用？

**A**: 
- 使用支持本地部署的模型（如 Ollama + 自定义 OpenAI 兼容接口）
- 知识库自动降级为 TF-IDF 本地引擎
- 文本分析、格式标准化、导出等功能均为纯本地实现

### Q9: 如何配置本地模型（Ollama）？

**A**:
1. 安装并启动 Ollama，拉取中文模型（如 `qwen2.5`）
2. 在 API 配置弹窗中选择「自定义」
3. Base URL 填写 `http://localhost:11434/v1`
4. API Key 填写 `ollama`（Ollama 默认无需真实 Key）
5. 测试连接后保存
