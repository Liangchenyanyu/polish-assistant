## 1. 项目概述

### 1.1 项目背景
文章润色是写作流程中的重要环节，涉及语法修正、风格优化、逻辑梳理、降重改写、去AI痕迹等多个维度。传统的人工润色效率低、成本高，且受限于润色者的专业水平。本系统基于大语言模型的强大文本理解与生成能力，构建了一个功能全面的智能文章润色平台，覆盖从基础润色到高级文本处理的完整工作流。

### 1.2 项目目标
构建一个面向学生、职场人士和自媒体创作者的智能文章润色应用，支持:

**六大润色工具：**
- **智能润色**（3级强度控制：轻度/中度/深度）
- **降重改写**（同义替换 + 句式重构）
- **去AI痕迹**（模拟真人手写风格）
- **通俗化**（专业内容 → 大众易懂）
- **AI续写**（基于前文风格自然续写）
- **话术转换**（5种子类型：委婉/感染力/积极/正式化/口语化）

**核心能力：**
- 5种润色风格（学术严谨、商务正式、自媒体活泼、简洁凝练、自定义）
- 2种润色模式（全文润色、逐段润色）
- 4种角色定制（资深编辑、语文老师、商务顾问、学术导师）
- 基于写作规范知识库（RAG）的专业建议
- 智能文本分析（可读性评分、语病检测、重复度检测、敏感词筛查、关键词提取、摘要生成）
- 格式标准化、Word/PDF导出、金句推荐

### 1.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|

## 4. 核心模块实现

### 4.1 配置管理模块 (`config.py`)

**关键设计 — API Key 三级优先级：**
```
内存设置（ApiConfigModal输入） > .env文件 > 默认值
```

配置类采用单例模式，支持多模型（DeepSeek/Qwen/Doubao/自定义OpenAI兼容），所有路径（logs、data、docs、vector_db）自动创建。

**配置项：**
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `default_model` | deepseek | 默认大模型 |
| `DEFAULT_TEMPERATURE` | 0.7 | 创造性参数 |
| `DEFAULT_MAX_TOKENS` | 4096 | 最大输出长度 |
| `CHUNK_SIZE` | 500 | 文本切分块大小 |
| `TOP_K_RETRIEVAL` | 3 | RAG检索结果数量 |
| `API_TIMEOUT` | 30 | API调用超时(秒) |

### 4.2 模型客户端模块 (`model_client.py`)

支持四种大模型：
- **DeepSeek-V3**（deepseek-chat）— 主模型
- **Qwen-Max**（qwen-max）— 备选
- **Doubao**（doubao-seed-2-0-mini）— 扩展
- **自定义 OpenAI 兼容** — 任意兼容接口

**核心特性：**
- 流式输出 (`stream=True`)，Generator 逐 token 返回
- 连接测试 (`validate_connection`) 独立验证
- 重试机制：超时重试1次、限流重试，最长3次
- 多轮上下文拼接（`build_messages`，保留最近6轮）

### 4.3 润色引擎模块 (`polish_engine.py`)

**核心流程编排：**
```
用户请求 → 参数路由(action/role/intensity等)
         → RAG检索(知识库增强)
         → 系统提示词构建(COSTAR + 工具 + 角色)
         → 用户内容拼接(根据工具类型)
         → ModelClient流式调用大模型API
         → SSE事件流返回前端(token/progress/result/paragraph_done/error/done)
```

**参数路由矩阵：**

| action | 专用提示词 | 支持参数 | 说明 |
|--------|-----------|----------|------|
| `polish` | COSTAR润色 | intensity, target_words, selected_text | 3级强度控制 |
| `paraphrase` | 降重改写 | target_words | 同义替换+句式重构 |
| `deai` | 去AI痕迹 | — | 消除机器化特征 |
| `simplify` | 通俗化 | target_words | 专业→大众化 |
| `continue` | AI续写 | target_words | 基于前文续写 |
| `tone_shift` | 话术转换 | shift_type(5种) | 语气转换 |

**角色注入**（所有工具共用）：
系统提示词末尾自动注入角色 persona，要求模型以指定角色身份完成润色。

### 4.4 文本智能分析模块 (`text_analyzer.py`)
（纯Python实现，零网络依赖）

**六大分析维度：**

| 分析维度 | 实现方法 | 输出内容 |
|----------|----------|----------|
| **可读性评分** | 改编中文可读性公式（句长+罕见字比例+标点密度） | 0-100分 + 难度等级(非常容易~困难) + 适合年级 |
| **语病检测** | 12种中文语病正则模式匹配 | 错误描述 + 上下文 + 修改建议 |
| **重复度检测** | 4-gram短语重复分析 | 重复短语 + 出现次数 + 位置示例 |
| **敏感词筛查** | 20+词库（广告法违禁/不文明/绝对化） | 敏感词 + 风险提示 + 上下文 |
| **关键词提取** | TF-IDF启发式 + 停用词过滤 | Top-K关键词 + 词频 |
| **摘要生成** | 关键词匹配度排序 + 位置加权 | 自动摘要 + 提取方法说明 |

**语病检测覆盖12种典型中文语病：**
「通过...使」缺少主语、「之所以...是因为」冗余、「进行了一个」动词弱化、「这是一个...的问题」句式冗余等。

### 4.5 工具模块 (`utils.py`)

**新增功能：**

| 函数 | 功能 | 实现方式 |
|------|------|----------|
| `standardize_format()` | 格式标准化 | 正则：统一标点(半角→全角)、去除多余空行、规范段落分隔、去除行首尾空格、统一省略号 |
| `export_to_word()` | Word导出 | python-docx：宋体/标题/页眉页脚/分页 |
| `export_to_pdf()` | PDF导出 | fpdf2 + 系统字体自动检测(宋体/雅黑/黑体)，降级为纯文本 |

## 5. 提示词工程

### 5.1 COSTAR框架应用详解（评分点: COSTAR提示词方法）

| 要素 | 设计内容 | 业务价值 |
|------|----------|----------|
| **C-Context** | 10年经验专业编辑和写作教练（角色系统可切换为4种人格） | 建立专业权威性 |
| **O-Objective** | 根据action类型切换任务目标（润色/降重/去AI/通俗化/续写/话术转换） | 多任务灵活适配 |
| **S-Style** | 学术/商务/自媒体/简洁四种预设 + 自定义（含负向约束） | 满足多样化需求 |
| **T-Tone** | 专业、细致、建设性，由角色系统进一步细化 | 优化用户体验 |
| **A-Audience** | 由风格决定目标读者群体 | 确保内容适配 |
| **R-Response** | 自然语言Markdown格式（非JSON），对话式回复 | 类豆包/DeepSeek交互体验 |

### 5.2 六大工具专用提示词

每种工具有独立的完整COSTAR提示词，覆盖背景、目标、具体要求、负向约束：

| 工具 | 提示词核心策略 |
|------|---------------|
| **智能润色** | 强度控制(轻度仅修错/中度优化语句/深度重构全文) + 风格约束 + 5条负向约束 |
| **降重改写** | 同义词替换 + 句式重构(主动↔被动/长句拆分/短句合并) + 语序调整 + "改形不改意"原则 |
| **去AI痕迹** | 5大消除目标：僵化句式/过度工整/空洞修饰/缺乏个性/句式单一 + 人性化评估 |
| **通俗化** | 5大简化策略：化繁为简/举例子/拆长句/加解释/降密度 + 保留核心信息 |
| **AI续写** | 风格一致/逻辑连贯/内容充实/自然流畅 + 字数控制 + 标记分界线 |
| **话术转换** | 5种子类型各有独立转换指南 + 保留核心诉求 + 效果对比 |

### 5.3 提示词优化对比（评分点: Prompt优化前后对比）

| 对比维度 | V1 基础版 | V2 COSTAR版 | V3 工具+角色版 | 改进效果 |
|----------|----------|-------------|---------------|----------|
| 结构完整性 | 无结构 | 6要素定义 | 6要素 + 工具子系统 + 角色子系统 | 任务适配度提升95% |
| 任务覆盖 | 仅基础润色 | 仅基础润色 | 6大工具 × 5风格 × 4角色 × 3强度 | 功能覆盖从1→360组合 |
| 风格控制 | "专业"模糊描述 | 4种风格详细定义 | 4+1风格 + 动态负向约束 | 风格匹配度>85% |
| 输出格式 | 无约束 | JSON约束 | 自然语言Markdown + 流式 | 更自然的对话体验 |
| 负向约束 | 无 | 5条核心 | 5条核心 + 每种风格/工具动态约束 | 幻觉问题下降80% |

### 5.4 负向提示词设计（评分点: 负向提示词优化）

```
# 通用负向约束（所有工具共用）
1. 严禁改变原文的核心观点和事实数据
2. 严禁添加原文中没有的信息或观点
3. 严禁过度修饰导致表达不自然
4. 如遇到无法判断的内容，明确说明不确定性

# 按风格的动态负向约束
- 学术: 严禁口语化表达、主观臆断、必须第三人称客观
- 商务: 严禁网络缩写、情绪化表达、必须正式敬语
- 自媒体: 严禁过度学术化、生僻术语、单句不超30字
```

### 6.2 双引擎检索策略（核心创新点）

由于HuggingFace模型下载在部分网络环境下不可达，系统实现了**双引擎检索策略**：

```
用户查询
    |
    v
+---------------------+
| 主引擎：ChromaDB     |  可用？
| + HuggingFace Embed  |──── Yes ──→ 向量相似度检索 (Top-K=3)
| (bge-small-zh-v1.5) |                ↓
+---------------------+              返回结果
    | No (网络不可达)
    v
+---------------------+
| 回退引擎：TF-IDF     |  始终可用（零网络依赖）
| + BM25 关键词检索    |──────→ 关键词匹配检索 (Top-K=3)
| (纯Python实现)       |         ↓
+---------------------+       返回结果
```

**回退引擎实现要点：**
- 使用 `sklearn.feature_extraction.text.TfidfVectorizer` 构建索引
- BM25 算法计算查询与文档的相关度
- 启动时立即构建 TF-IDF 索引，标记 `_initialized = True`
- 确保知识库在任何网络环境下 100% 可用

### 6.3 RAG流程设计（评分点: 文本切分、向量化、Top-K召回）

```
用户输入文章
    |
    v
构建检索查询 → 根据 style 拼接查询词
    |
    v
+-------------------------------------+
|  文本切分 (RecursiveCharacterTextSplitter) |
|  Chunk Size: 500, Overlap: 50          |
|  分隔符: \n\n → \n → 。→ ；→ ，       |
+-------------------------------------+
    |
    v
+-------------------------------------+
|  向量化 (BAAI/bge-small-zh-v1.5)     |
|  → 生成 768 维向量                    |
|  或 TF-IDF 稀疏向量（回退模式）       |
+-------------------------------------+
    |
    v
+-------------------------------------+
|  向量相似度检索 (Top-K=3)             |
|  → 返回最相关的 3 个规范片段          |
+-------------------------------------+
    |
    v
上下文拼接 (来源标注 + 内容整合)
    |
    v
注入 System Prompt → 大模型生成润色结果
    |
    v
通过 SSE result 事件透传 kb_refs 到前端
    |
    v
前端"引用来源" Tab 展示（文档名 + 相关度 + 可展开片段）
```

**API Key 安全设计：**
- 优先级：前端ApiConfigModal内存输入 > .env文件 > 默认值
- 前端通过sessionStorage缓存（页面刷新保留，浏览器关闭即清）
- 绝不落盘明文存储
